# 🎨 Design System — Idea Validator

## Дизайн-философия

**Субъект:** Веб-приложение для валидации стартап-идей через AI-анализ  
**Аудитория:** Предприниматели, инвесторы, основатели (люди, принимающие риски и решения)  
**Уникальный элемент:** Номерованные разделы (01-08) как визуальная "лента анализа"

---

## 🎨 Цветовая палета

### Темный режим (основной)
```
--deep-navy:        #0F172E  (фон)
--bright-emerald:   #00D084  (акцент, CTA, валидация)
--warm-amber:       #FF9F43  (вторичный, риски/внимание)
--light-gray:       #E5E7EB  (текст на темном)
--surface:          #1A1F35  (карточки)
--surface-alt:      #252D47  (поднутые поверхности)
--text-secondary:   #A1A5B0  (secondary text)
--border:           #2D3447  (разделители)
```

### Светлый режим
Система автоматически переключается на светлый режим если браузер предпочитает `prefers-color-scheme: light`.

**Логика выбора палеты:**
- **Deep Navy** — серьёзность, технологичность, доверие
- **Bright Emerald** — валидация, успех, энергия (контрастирует с тёмным фоном)
- **Warm Amber** — риски, предупреждения, вторичные акценты
- Остальное — нейтральные грeys для иерархии и читаемости

## 🔤 Типография

### Гарнитуры
- **Display:** `Inter Bold 700-800` — заголовки, современно и уверенно
- **Body:** `Inter Regular 400` — основной контент, чистота и читаемость
- **Utility:** `Fira Code 400` — данные, цифры, метаданные (техничность)

### Масштаб
```
h1:   3.0rem, font-weight: 700, letter-spacing: -0.02em
h2:   1.5rem, font-weight: 700
h3:   1.8rem, font-weight: 700, letter-spacing: -0.01em
body: 1.0rem, font-weight: 400, line-height: 1.8
```

### Логика
- Градиент в h1 (зелёный → серый) для визуального интереса
- Чистый Inter для основного контента для максимальной читаемости
- Fira Code для номеров разделов (01, 02, 03...) — напоминает технологичность

## 🔑 Ключевые компоненты

### Сигнатура: Numbered Sections
```html
<div class="section" data-number="01">
    <h3>Market Size & Trends</h3>
    <p>...</p>
</div>
```

**Функция:**
- Номер (01-08) отображается слева вверху каждой секции
- Помогает навигировать через 8 разделов анализа
- Создаёт визуальную "ленту" анализа
- Работает как якорь для идентификации разделов

### Кнопки (CTA)
```
background: var(--bright-emerald)
color: var(--deep-navy)
padding: 14px 28px
border-radius: 12px
font-weight: 700
transition: all 0.3s ease
```

**Поведение:**
- Hover: светлее зелёный, поднять на 2px, тень
- Active: вернуть на место
- Disabled: серый (--text-secondary)

### Карточки
```
background: var(--surface)
border: 1px solid var(--border)
border-radius: 16px
transition: all 0.3s ease
```

**Поведение:**
- Hover: граница становится зелёной, тень с зелёным свечением

### Рейтинг (звёзды)
- Размер: 2.5rem
- Цвет при наведении: --warm-amber
- Эффект: scale(1.1), text-shadow свечение
- Интерактивность: hover показывает предпросмотр рейтинга

## ✨ Микровзаимодействия

### Анимации
```css
fadeIn:    0.5s ease-out (opacity + translateY)
slideDown: 0.3s ease-out (для ошибок и успеха)
spin:      0.8s linear infinite (спиннер)
```

### Переходы
- Основное время: `0.3s ease`
- Быстрые: `0.2s ease` (input focus)
- Спиннер: `0.8s linear` (постоянная)

### Accessibility
- `:focus-visible` имеет зелёный outline (2px)
- Поддержка `prefers-reduced-motion`
- Все интерактивные элементы имеют видимый focus state
- Звёзды имеют `aria-label` для screen readers

## 🌓 Темный режим (проактивный выбор)

Темный режим установлен как **основной** в дизайне (не опция). Это решение:
- Подходит современной web-аудитории
- Улучшает восприятие ярких акцентов (зелёный выглядит ярче на тёмном)
- Снижает нагрузку на глаза при длительном чтении
- Выглядит premium и инновационно

Браузер может переопределить через `prefers-color-scheme: light`, но дизайн оптимизирован для тёмного.

## 🎯 Дизайн-решения и обоснование

### 1. **Bright Emerald как основной акцент**
- **Выбор:** Вместо традиционного blue
- **Обоснование:** Зелёный означает валидацию/успех, что соответствует идее приложения (валидация идеи). На тёмном фоне выглядит ярко и современно.

### 2. **Номерованные разделы (01-08)**
- **Выбор:** Вместо просто названий
- **Обоснование:** Помогает пользователю понять, что это структурированный процесс из 8 шагов. Создаёт ощущение прогресса.

### 3. **Темный режим как основной**
- **Выбор:** Не светлый + опция тёмный
- **Обоснование:** Современный стандарт для tech-ориентированной аудитории. Меньше стресса для глаз. Премиум ощущение.

### 4. **Inter + Fira Code**
- **Выбор:** Специфичная комбинация типографии
- **Обоснование:** Inter — современный, читаемый, используется в Silicon Valley. Fira Code — напоминает код/технологии. Вместе передают "tech startup" ощущение.

### 5. **Градиент в h1**
- **Выбор:** Зелёный → Серый градиент текста
- **Обоснование:** Добавляет визуальный интерес к главному заголовку, не переусложняя остальное. Один смелый выбор в начале.
