# Спека: редизайн лендинга (тёмный продающий)

**Дата:** 2026-07-14
**Статус:** черновик, ждёт одобрения
**Направление (согласовано):** тёмная премиум-эстетика (Deep Navy + Emerald), продающая **структура** ShipFit — НЕ светлый вид, тёмная тема остаётся по умолчанию. Только фронт (index.html + style.css), бэкенд не трогаем.

---

## Зачем
Текущий лендинг = голый hero + форма. Продающей логики нет. Берём у ShipFit то, что реально конвертит (крупный hero с оффером, «как работает», «что получишь», отстройка), но в своей тёмной эстетике. Цель — понятность оффера и продающая структура для теста спроса, без смены визуального языка.

## Заголовок hero (согласован)
- **H1:** «Не строй то, что <span class="accent">никому не нужно</span>.» — акцент градиентом (Emerald→Amber) на «никому не нужно».
- **Подзаголовок:** «Честный AI-разбор идеи за 30 секунд — прежде чем потратишь полгода.»
- Угол отстройки: боль основателя + честность против лести ChatGPT (не калька ShipFit «kill/win»).

## Структура страницы (сверху вниз)
1. **HERO** — H1 + подзаголовок + **существующая форма ввода** (id `ideaForm`/`idea`) прямо тут + бейджи + чипы-примеры + CTA «Анализировать идею». Форма НЕ уезжает вниз — низкое трение остаётся.
2. **БЛОК «Как это работает»** — 3 шага в ряд (на мобильном — в колонку):
   1. Опиши идею одним абзацем.
   2. AI разбирает 8 аспектов через проверенные фреймворки (The Mom Test, JTBD, TAM/SAM/SOM…).
   3. Честный вердикт GO / PIVOT / KILL + что проверить дальше.
3. **БЛОК «Что ты получишь»** — сетка из 8 карточек с номерами 01–08 (Market Size & Trends, Target Customer, Pain Points, Solution, Competition, MVP Scope, Pricing & Business Model, Key Risks). Показывает глубину до ввода идеи.
4. **БЛОК «Почему не просто спросить ChatGPT»** — отстройка: «ChatGPT хвалит любую идею и идёт дальше. Idea Validator режет правду через проверенные фреймворки. Бесплатно, без регистрации, твои идеи никуда не публикуются.»
5. Ниже — существующее без изменений: результат анализа (`analysisCard`), форма отзыва, отзывы, waitlist.

## Реализация (технически)
- **Файлы:** только `frontend/templates/index.html` (разметка новых секций) + `frontend/static/style.css` (стили). JS и бэкенд не трогаем.
- **Переменные:** переиспользуем существующие (`--deep-navy`, `--bright-emerald`, `--warm-amber`, `--surface`, `--border`). Amber наконец идёт в дело — в градиент-акценте.
- **Hero H1:** крупный (~3.5rem desktop / 2rem mobile), `.accent` — `linear-gradient(135deg, emerald, amber)` через `background-clip: text`.
- **Блоки:** секции-обёртки + CSS grid (3 колонки для шагов, 4×2 или auto-fit для 8 карточек), карточки в стиле существующих `.card`. На мобильном — 1 колонка.
- **КРИТИЧНО — не сломать JS:** сохранить все существующие id (`ideaForm`, `idea`, `analyzeBtn`, `charCount`, `analysisCard`, `analysisResult`, `reviewForm`, `waitlistForm`, `.example-chip` и т.д.). Редизайн — это перекомпоновка разметки вокруг них + новые статические секции, а не переписывание логики.
- **Responsive:** все сетки схлопываются в колонку ≤768px (media-query уже есть, дополняем).

## Что НЕ делаем (YAGNI)
- Не светлую тему и не переключатель тем.
- Не FAQ, не прайсинг, не блоки с фейковыми метриками/отзывами (спрос не доказан, врать нельзя).
- Не трогаем бэкенд, БД, JS-логику анализа/отзывов/waitlist.
- Не добавляем зависимости.

## Риски
- Единственный реальный риск — задеть id/классы, на которых висит JS. Митигация: после правок прогнать полный флоу в браузере (ввод идеи → анализ → отзыв → waitlist → клик чипа) перед деплоем.

## Проверка (definition of done)
- Все 3 новых блока рендерятся, тёмная тема, градиент-акцент виден.
- Форма и весь существующий функционал работают (анализ, чипы, отзывы, waitlist).
- Mobile: секции в колонку, ничего не наезжает.
- Проверено в браузере локально → деплой `railway up` → проверка на проде.
