# Пересмотренный порядок работ (Idea Validator)

**Дата:** 2026-07-14
**Контекст:** после разбора конкурентов (ValidMyIdea, ValidateIdea, IdeaProof, **ShipFit**) пересмотрели приоритеты. Вывод: ядро ценности — **качество основного AI-анализа**, а не количество фич. Сначала прокачиваем ядро, «План валидации» отодвигаем.

---

## Ключевой вывод по конкурентам

**ShipFit (shipfit.ai)** — сильнейший прямой конкурент. Делает ровно то, что мы считали «незанятой нишей»: мост идея→действие (smoke-test kit, The Mom Test, launch plan) + vibe-code экспорт (`CLAUDE.md`, `.cursorrules`) за $5–10 разово. Два его козыря:
- **55 фреймворков** (Mom Test, JTBD, 7 Powers, Van Westendorp, TAM/SAM/SOM…) — по сути вопрос **промпта**, копируется концептуально дёшево.
- **«Реальные данные, не AI-slop»** — живой скрапинг конкурентов/цен/отзывов из Trustpilot/G2/Reddit/App Store. Дорого и сложно (нужны инструменты web search), не просто промпт.

**Отстройка соло-продукта** (не меняем): бесплатно + мгновенно + приватно (идеи не публикуются) + простота без 9 обязательных шагов. По «реальным данным» и 55 фреймворкам их не догнать — и не надо.

## Порядок работ

### Шаг 0 — Оплатить Anthropic API (~$5) — БЛОКЕР, но перед финалом
Пока баланса нет, прод отдаёт заглушку. **Оплата откладывается до финала** (решение пользователя): отработку промпта ведём вручную через claude.ai, а API включаем только когда промпт готов и пора запускать живой анализ на проде.

### Шаг 1 — Прокачать основной промпт `content/analysis_prompt.txt` (СЕЙЧАС)
Ядро продукта. Что делаем:
- Вписать **фреймворки по имени** в промпт (The Mom Test, Jobs-to-be-Done, TAM/SAM/SOM, Van Westendorp для цены, Unit Economics LTV/CAC) — чтобы каждый раздел был обоснован методом, а не «AI подумал». Это «фреймворки как у ShipFit» почти бесплатно.
- Потребовать конкретику и источники, резкий честный тон (наш голос — «The Roast»-стиль без розовых соплей).
- Сохранить структуру 8 разделов + FINAL VERDICT (совместимо с текущим фронтом).

**Workflow отработки (важно):** промпт итерируем **вручную через claude.ai**, НЕ через API.
1. Готовлю текст промпта →
2. Пользователь вставляет в чат claude.ai, гоняет на 3–5 разных идеях (B2B SaaS, B2C-приложение, маркетплейс…) →
3. Смотрим слабые места, правлю промпт →
4. Повторяем, пока выдача не станет стабильно хорошей →
5. Финальную версию кладём в `analysis_prompt.txt`.
API оплачиваем (Шаг 0) только перед включением живого анализа на проде.

### Шаг 2 — Реальные данные через web search (ПОЗЖЕ)
Подключить Claude web search в пайплайн анализа, чтобы подтягивать настоящих конкурентов, цены и жалобы (как ShipFit/IdeaProof). Средние трудозатраты. Именно это отделяет «серьёзный продукт» от «ещё один GPT-врапер». Требует оплаченного API.

### Шаг 3 — Фича «План валидации» (ПОСЛЕ ядра)
Бесплатный генератор плана действий (мост к реальным интервью). Детальная спека: [05_VALIDATION_PLAN_SPEC.md](05_VALIDATION_PLAN_SPEC.md). Гибрид шаблон+AI, потом платный gate на Lemon Squeezy.

### Шаг 4 — Редизайн лендинга (опционально, по желанию)
ShipFit-стиль нравится пользователю: светлая тема, крупный hero с одним оффером, чипы-примеры идей, «первый вердикт бесплатно без карты». Отдельная задача, не мелкая. Наш тёмный технический вид — другой жанр («инструмент» vs «продукт для продажи»); их лендинг лучше конвертит новичка.
